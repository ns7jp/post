# post コード読解ガイド

このドキュメントは、スタイリッシュ掲示板アプリの各ファイルが「何を担当しているか」「どの順番で読むと理解しやすいか」を初学者向けに整理したものです。コード内にもコメントを追記していますが、最初はこのガイドで全体像をつかんでから各ファイルを読むと、PHP と MySQL のつながりが見えやすくなります。

---

## 1. アプリ全体の流れ

このアプリは、会員登録してログインしたユーザーが投稿や返信を行える掲示板です。大きな流れは次の通りです。

```text
ブラウザ
  ↓ フォーム入力
PHP ファイル
  ↓ 入力値の受け取り・チェック
MySQL データベース
  ↓ members / posts テーブルに保存・取得
PHP が HTML を作る
  ↓
ブラウザに画面表示
```

`login.php` や `input.php` は画面を表示するファイル、`check.php` や `write.php` は送信されたデータを処理するファイルです。画面ファイルと処理ファイルが分かれている点が、このアプリを読むときの大きなポイントです。

---

## 2. 最初に読むおすすめ順

1. `README.md`
   - アプリ概要、ファイル構成、起動に必要なものを確認します。
2. `db.example.php`
   - DB 接続の考え方を確認します。実際の接続情報は `db.php` に作成します。
3. `login.php` → `check.php` → `logout.php`
   - ログイン画面、本人確認、ログアウトの流れを追います。
4. `input.php` → `comfirm.php` → `regist.php`
   - 新規会員登録の入力、確認、保存の流れを追います。
5. `index.php` → `write.php`
   - 投稿一覧の表示と新規投稿保存の流れを追います。
6. `reply.php` → `write2.php` → `index2.php`
   - 返信投稿と、親投稿の下に返信を表示する流れを追います。
7. `delete.php` / `error.php`
   - 削除処理とエラー画面のような補助ファイルを確認します。
8. `style.css`
   - 最後に見た目の調整を確認します。

---

## 3. ファイル別の説明

### `README.md`

リポジトリ全体の説明書です。アプリの概要、デザイン特徴、ファイル構成、セットアップ手順が書かれています。

初学者が見るポイント:
- どのファイルが画面用で、どのファイルが処理用か
- `db.example.php` と実際に作る `db.php` の違い
- `members` テーブルと `posts` テーブルの役割

---

### `DATABASE_SETUP.md`

MySQL のテーブル構成やセットアップ方法を説明する補助ドキュメントです。

初学者が見るポイント:
- `members` は会員情報を保存するテーブル
- `posts` は投稿と返信を保存するテーブル
- `reply_post_id` が `0` なら通常投稿、値が入っていれば返信
- 現在の PHP コードでは `password_hash()` / `password_verify()` を使っているため、パスワードはハッシュ化前提で読む

---

### `db.example.php`

データベース接続設定の見本です。実際にアプリが読み込むのは `db.php` なので、このファイルをコピーして `db.php` を作ります。

主な処理:
- `new PDO(...)` で MySQL に接続する
- 接続できない場合は `catch` でエラー処理する
- `PDO::ATTR_ERRMODE` を `PDO::ERRMODE_EXCEPTION` にして、SQL エラーに気づきやすくする

初学者が見るポイント:
- DB 接続情報にはパスワードなどの秘密情報が含まれるため、公開用には `db.example.php` を置く
- `charset=utf8mb4` は日本語や絵文字を扱いやすくする設定
- 各 PHP ファイルの `require("db.php")` は、この設定を読み込むための行

---

### `login.php`

ログイン画面を表示するファイルです。

主な処理:
- `session_start()` でセッションを開始する
- CSRF 対策用のトークンを作り、セッションに保存する
- メールアドレスとパスワードの入力フォームを表示する
- 送信先を `check.php` にする

初学者が見るポイント:
- このファイル自体はログイン判定をせず、入力フォームを作る役割
- hidden の `token` は画面には見えないが、`check.php` で照合される
- フォームの `name="id"` や `name="pass"` が、`check.php` の `$_POST` と対応している

---

### `check.php`

ログインフォームから送られた情報を確認する処理ファイルです。

主な処理:
- `$_POST` からメールアドレス、パスワード、CSRF トークンを受け取る
- セッション内のトークンと送信されたトークンを比較する
- `members` テーブルにメールアドレスが存在するか確認する
- `password_verify()` でパスワードを照合する
- 成功時は `$_SESSION["user_id"]` に会員 ID を保存する

初学者が見るポイント:
- ログイン状態は「セッションに user_id があるか」で判断している
- SQL は `prepare()` と `execute()` で安全に値を渡している
- `COUNT` で存在確認し、その後にユーザー情報を取得する2段階の流れ

---

### `logout.php`

ログアウト処理だけを担当するファイルです。

主な処理:
- セッションを開始する
- `unset($_SESSION["user_id"])` でログイン情報を消す
- `login.php` にリダイレクトする

初学者が見るポイント:
- ログアウトは画面表示ではなく、ログイン状態を消す処理
- `session_unset()` や `session_destroy()` との違いもコメントで説明している
- 状態変更後は `header("Location: ...")` で別ページに移動する

---

### `input.php`

新規会員登録の入力フォームを表示するファイルです。

主な処理:
- 名前、メールアドレス、パスワード、プロフィール画像の入力欄を表示する
- 画像アップロードのため、`form` に `enctype="multipart/form-data"` を付ける
- 入力内容を `comfirm.php` に送る

初学者が見るポイント:
- ファイルアップロードを含むフォームでは `enctype` が必須
- `name` 属性が、次の `comfirm.php` で受け取るキーになる
- このファイルは DB 保存をせず、入力を次の確認画面へ渡すだけ

---

### `comfirm.php`

登録内容の確認画面です。ファイル名は `confirm` ではなく `comfirm` になっていますが、既存リンクに合わせてこの名前で使われています。

主な処理:
- `input.php` から送られた名前、メール、パスワードを受け取る
- アップロードされた画像を `image/` フォルダへ保存する
- 画面に登録内容を表示する
- hidden input に値を入れて `regist.php` へ渡す

初学者が見るポイント:
- 確認画面では、画面に表示する値と次へ送る値の両方を扱う
- hidden input は画面に見えないが、次の処理へ値を渡すために使う
- 画像ファイルは `$_FILES` で受け取り、`move_uploaded_file()` で保存する

---

### `regist.php`

会員登録を実際に DB に保存する処理ファイルです。

主な処理:
- `comfirm.php` から hidden input で送られた値を受け取る
- パスワードを `password_hash()` でハッシュ化する
- `members` テーブルに `INSERT` する
- 保存後は `login.php` にリダイレクトする

初学者が見るポイント:
- パスワードはそのまま保存せず、ハッシュ化して保存する
- `INSERT INTO members SET ...` で新しい会員行を追加している
- 登録後にログイン画面へ戻すことで、登録とログインを分けている

---

### `index.php`

掲示板のメイン画面です。新規投稿フォームと投稿一覧を表示します。

主な処理:
- セッションに `user_id` があるか確認し、未ログインなら `login.php` へ移動する
- ログイン中ユーザー名を DB から取得する
- 投稿フォームを表示し、送信先を `write.php` にする
- `members` と `posts` を結合して投稿一覧を取得する
- 投稿本文、投稿者名、画像、日時、削除リンク、返信リンクを表示する

初学者が見るポイント:
- ログインチェックはページの最初に行う
- `while` で DB の結果を1件ずつ取り出して HTML を作っている
- `nl2br()` で改行を HTML の `<br>` に変換している
- 自分の投稿だけ削除リンクを表示する条件分岐がある

---

### `write.php`

新規投稿を DB に保存する処理ファイルです。

主な処理:
- セッションから投稿者の会員 ID を取得する
- `$_POST["message"]` から投稿本文を取得する
- `posts` テーブルに `message`、`member_id`、`created` を保存する
- 保存後は `index.php` に戻る

初学者が見るポイント:
- 画面表示ではなく保存処理が中心なので、短いファイルになっている
- 通常投稿は `reply_post_id` を指定しないため、DB 側の初期値 `0` が使われる
- 投稿者はフォームからではなくセッションから取得する

---

### `reply.php`

返信を書く画面です。

主な処理:
- URL の `?id=...` から返信元の投稿 ID を受け取る
- 返信元の投稿を DB から取得して表示する
- 返信フォームを表示する
- hidden input で返信元 ID を `write2.php` に渡す

初学者が見るポイント:
- URL の値は `$_GET` で受け取る
- 投稿本文の表示には `htmlspecialchars()` と `nl2br()` を使う
- 返信元 ID はユーザーに見せなくても必要なので hidden input に入れる

---

### `write2.php`

返信投稿を DB に保存する処理ファイルです。

主な処理:
- セッションから返信者の会員 ID を取得する
- `$_POST["message"]` から返信本文を取得する
- `$_POST["id"]` から返信元投稿 ID を取得する
- `posts` テーブルに `reply_post_id` 付きで `INSERT` する
- 保存後は `index.php` に戻る

初学者が見るポイント:
- `write.php` とほぼ同じだが、`reply_post_id` を保存する点が違う
- 同じ `posts` テーブルで通常投稿と返信を管理している
- 通常投稿と返信の違いは、`reply_post_id` が `0` かどうか

---

### `index2.php`

返信を親投稿の下にぶら下げて表示する別バージョンの一覧ページです。

主な処理:
- 親投稿、つまり `reply_post_id = 0` の投稿だけを先に取得する
- 親投稿ごとに返信数を調べる
- 返信がある場合、その親投稿への返信を取得して下に表示する

初学者が見るポイント:
- `index.php` は全投稿を新しい順に並べる
- `index2.php` は親投稿と返信を分けて表示する
- 同じデータでも、SQL と表示方法を変えると見え方が変わる

---

### `delete.php`

投稿を削除する処理ファイルです。

主な処理:
- URL の `?id=...` から削除対象の投稿 ID を受け取る
- `posts` テーブルから対象 ID の行を `DELETE` する
- 削除後は `index.php` に戻る

初学者が見るポイント:
- 削除対象 ID は URL から来るので、必ず安全に扱う必要がある
- `prepare()` と `execute()` で SQL に値を渡している
- 実運用では「自分の投稿かどうか」の権限チェックも必要

---

### `error.php`

ログイン失敗時のエラー画面です。

主な処理:
- エラーメッセージを表示する
- `login.php` に戻るリンクを表示する
- 共通 CSS を読み込んで見た目をそろえる

初学者が見るポイント:
- PHP 処理がほぼなく、HTML と CSS だけのページ
- 処理ファイルからエラー時に移動してくる「結果表示ページ」として読む

---

### `style.css`

アプリ全体の見た目を担当する CSS ファイルです。

主な処理:
- 全体の背景、文字、余白を設定する
- 投稿フォーム、投稿カード、ボタン、エラーメッセージの見た目を整える
- ホバー時の動きやアニメーションを定義する
- `@media` でスマートフォン表示に対応する

初学者が見るポイント:
- HTML の `class` と CSS の `.class名` が対応している
- `display: flex` や `gap` で横並びや間隔を調整している
- `box-shadow`、`border-radius`、`linear-gradient` でカード型 UI を作っている
- レスポンシブ対応はファイル後半の `@media` を読むと分かりやすい

---

## 4. 代表的な処理の追い方

### ログイン

```text
login.php
  ↓ フォーム送信
check.php
  ↓ CSRF トークン確認
members テーブルでメールアドレスを検索
  ↓ password_verify()
$_SESSION["user_id"] に会員IDを保存
  ↓
index.php
```

### 会員登録

```text
input.php
  ↓ 入力内容と画像を送信
comfirm.php
  ↓ 内容確認、画像保存、hidden input で値を引き継ぎ
regist.php
  ↓ password_hash()、members テーブルに INSERT
login.php
```

### 新規投稿

```text
index.php
  ↓ 投稿フォーム送信
write.php
  ↓ セッションの user_id と本文を取得
posts テーブルに INSERT
  ↓
index.php
```

### 返信

```text
index.php の返信リンク
  ↓ ?id=投稿ID
reply.php
  ↓ 返信本文と返信元IDを送信
write2.php
  ↓ reply_post_id 付きで posts テーブルに INSERT
index.php / index2.php
```

### 削除

```text
index.php / index2.php の削除リンク
  ↓ ?id=投稿ID
delete.php
  ↓ posts テーブルから DELETE
index.php
```

---

## 5. 学習時に意識するとよいこと

- 画面表示ファイルと処理ファイルの役割を分けて読む
- フォームの `name` 属性と、受け取り側の `$_POST["..."]` を対応させる
- URL の `?id=...` と、受け取り側の `$_GET["id"]` を対応させる
- DB に値を渡すときは `prepare()` / `execute()` を使う
- 画面に文字を表示するときは `htmlspecialchars()` を使う
- ログイン状態は `$_SESSION["user_id"]` で管理している
- 投稿と返信は同じ `posts` テーブルに入り、`reply_post_id` で区別している
